# 行业白皮书生成器

## 快速开始

当用户提供行业素材时，自动执行以下流程：

1. **分析素材** → 提取核心事实、数据和观点
2. **构建骨架** → 映射到五段式结构
3. **深度扩写** → 运用扩写策略扩展3-5倍篇幅
4. **风格润色** → 检查词汇和句式复杂度
5. **补充数据** → 必要时调用搜索脚本补充信息
6. **构建产业图谱** → 提取企业信息，按上中下游分类，联网补充缺失信息

## 核心规则（最高优先级）

### 内容体量控制

- **篇幅要求**：每个二级标题下至少 **500-1000字**
- **禁止清单体**：严禁bullet points，必须转化为连贯段落
- **扩写策略**：
  - 背景铺垫（全球经济、国家战略、技术周期）
  - 数据三维分析（数值→驱动力→未来趋势）
  - 逻辑填充（利用行业常识推演扩展）

### 五段式结构（严格执行）

```
头部信息（标题、副标题、日期）
├── 前言（300-500字综述）
├── 一、总体态势（宏观政策、市场规模、发展阶段）
├── 二、关键技术趋势（技术突破、数实融合）
├── 三、产业发展趋势（竞争格局、产业链整合、出海）
├── 四、行业面临的问题（供给侧vs需求侧矛盾）
├── 五、对行业发展的对策建议（排比句气势）
└── 附录：产业图谱（上中下游企业分类与信息阐释）
```

### 语言规范

- **词汇升维**：增长→"迈上新台阶"；合作→"深度耦合"；困难→"深层次结构性矛盾"
- **句式特征**：使用长难句，包含"虽然...但随着...正经历从...向...的路径演进"
- **数据融入**：严禁孤立展示数据，必须嵌入长句作为论据

## 工具和资源

### 数据补充脚本

当素材不足时，使用搜索脚本补充：

```bash
# 搜索行业最新数据
scripts/search_industry_data.py --industry "<行业名称>" --topic "<主题>"

# 搜索政策背景
scripts/search_policy_context.py --keyword "<关键词>"
```

### 企业信息联网搜索

产业图谱构建时，对信息不足的企业进行联网搜索补充：

```bash
# 搜索企业基本信息
scripts/search_company_info.py --company "<企业名称>" --industry "<行业>"

# 搜索企业产品与生态位
scripts/search_company_products.py --company "<企业名称>" --segment "<产业链环节>"
```

**企业信息补充标准**：
- 基本情况：成立时间、总部地点、企业规模、融资/上市情况
- 主要产品：核心产品线、技术特点、市场定位
- 行业生态位：产业链环节（上游原材料/中游制造/下游应用）、竞争优势
- 补充时机：仅有企业名称且无详细信息，或关键信息缺失

## 输出流程

1. 读取素材，提取关键信息点
2. 如需补充数据，调用 `scripts/search_industry_data.py`
3. 按 [STRUCTURE.md](references/STRUCTURE.md) 构建骨架
4. 使用 [VOCABULARY.md](references/VOCABULARY.md) 和 [STYLE_GUIDE.md](references/STYLE_GUIDE.md) 进行深度扩写
5. 参照 [DATA_ANALYSIS.md](references/DATA_ANALYSIS.md) 分析数据
6. **构建产业图谱附录**：
   - 提取素材中所有企业名称
   - 按产业链环节分类（上游/中游/下游）
   - 对信息不足的企业调用联网搜索补充
   - 生成企业信息列表阐释表格
7. 按 [OUTPUT_TEMPLATE.md](assets/OUTPUT_TEMPLATE.md) 格式输出
8. 自检：确认每节≥500字，无bullet points
