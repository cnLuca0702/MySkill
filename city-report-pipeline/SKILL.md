# 城市调研报告完整流水线技能

## 技能概述

本技能是一个编排型（Orchestration）技能，通过顺序调用两个子技能，完成从数据收集到报告输出的完整闭环：

1. **第一阶段**：调用 `city-research-deep` 技能，完成系统化数据收集与9章初稿撰写
2. **第二阶段**：调用 `multi-search-engine` 技能，对初稿逐章进行全面深度充实完善
3. **第三阶段**：将最终报告同时输出为 `.md` 和 `.docx` 两种格式

**核心约束：**
- 充实完善过程中，文字风格严格保持与初稿一致（专业、数据化、无列表符号、无表格、无emoji）
- 字数不设上限，以数据充分、论证完整为准
- 数据必须标注来源和时间节点

---

## 触发条件

当用户提出以下类型请求时触发本技能：
- 帮我深度调研某个城市/城区
- 输出完整的城市发展调研报告
- 生成专业的城市分析报告（含文件导出）
- 对某城市进行系统性研究并导出报告

---

## 第一阶段：调用 city-research-deep 生成初稿

### 1.1 执行说明

完整执行 `city-research-deep` 技能的全部工作流程，包括：

- 使用17个搜索引擎系统化收集数据（行政区划、人口、经济、交通、产业、房产、公共服务、城市更新）
- 对所有关键数据进行多源交叉验证，标注数据来源和时间节点
- 严格按照9章固定框架撰写完整初稿
- 每段不少于150字，专业文字风格，无列表符号、无表格、无emoji

### 1.2 初稿完成标志

初稿须满足以下全部条件方可进入第二阶段：

- [ ] 9个章节完整（区域概况→交通圈→人口→经济→城市更新→房地产→公共服务→城市定位→发展展望）
- [ ] 各章节均有具体数据支撑
- [ ] 关键数据（人口、GDP、房价、教育床位等）均已标注来源和年份
- [ ] 无列表符号、无表格、无emoji、全程阿拉伯数字

---

## 第二阶段：调用 multi-search-engine 深度充实初稿

### 2.1 执行说明

使用 `multi-search-engine` 技能提供的17个搜索引擎，对初稿逐章进行深度扩充。搜索方式为直接URL访问，无需API Key：

```
web_fetch({"url": "https://www.google.com/search?q={关键词}"})
web_fetch({"url": "https://www.baidu.com/s?wd={关键词}"})
```

### 2.2 逐章充实清单

对初稿每一章执行以下操作，直至无新增有效信息为止：

#### 第1章：区域概况与战略定位
- 补充城市在国家战略体系中的具体定位文件依据（政策原文、规划文件）
- 充实地理区位的战略价值分析，扩展历史沿革的叙述深度
- 搜索引擎重点：百度、必应中国、新华网、人民网、政府官网

```
{城市名称} 国家战略 定位 规划 site:gov.cn
{城市名称} 历史沿革 建城 区划调整 filetype:pdf
{城市名称} 城市群 战略 节点 枢纽 2024
```

#### 第2章：两小时交通圈与区域协同分析
- 补全交通圈内所有城市的人口、GDP、产业结构数据
- 充实各城市与调研城市的产业互补关系和协同发展案例
- 搜索引擎重点：百度、必应国际、12306官网、交通运输部官网

```
{城市名称} 高铁 班次 时刻表 2024 2025
{城市名称} 两小时交通圈 覆盖 城市 人口
{区域} 产业协同 互补 合作
```

#### 第3章：人口结构与社情特征
- 补充历次人口普查对比数据，展示人口结构演变趋势
- 充实民族文化特色、流动人口结构、人才政策细节
- 搜索引擎重点：政府统计局、聚汇数据、红黑人口库

```
{城市名称} 第七次人口普查 第六次 对比 变化
{城市名称} 流动人口 外来人口 人才引进
{城市名称} 民族 文化 宗教 特色
```

#### 第4章：经济发展与产业布局
- 补充重点企业名录、产业链上下游关系、龙头企业产值
- 充实各产业的技术创新成果、专利数据、产业园区详情
- 搜索引擎重点：百度、政府官网、发改委、工信部

```
{城市名称} 重点企业 龙头 产值 2024
{城市名称} 产业园 入驻企业 规模
{城市名称} 专利 技术 创新 成果
{城市名称} 统计公报 2024 filetype:pdf site:gov.cn
```

#### 第5章：城市更新与空间优化
- 补充具体改造项目名称、地点、投资额、完工情况
- 充实城市基础设施现状与薄弱环节的具体表现
- 搜索引擎重点：住建局官网、百度、微信

```
{城市名称} 老旧小区 改造 项目 投资 2023 2024
{城市名称} 城中村 改造 置换
{城市名称} 城市更新 基础设施 海绵城市
```

#### 第6章：房地产市场与居住生态
- 补充各主要片区详细房价数据、代表性楼盘名称
- 充实居住生态内容：公园、湿地、商圈、学区具体信息
- 搜索引擎重点：中国房价行情、安居客、房天下

```
web_fetch({"url": "https://m.creprice.cn/district/{区代码}.html?city={城市代码}"})
web_fetch({"url": "https://m.anjuke.com/{城市拼音}/trendency/{区域代码}/"})
{城市名称} 各片区 均价 楼盘 2024 2025
```

#### 第7章：公共服务与民生保障
- 补充学校名单、等级、入学数据、医院等级和床位数详情
- 充实养老、文化、体育等公共设施的具体布局和覆盖情况
- 搜索引擎重点：教育局官网、卫健委官网、百度

```
{城市名称} 学校 名单 等级 招生 2024
{城市名称} 医院 三甲 二甲 床位 数量
{城市名称} 养老机构 数量 床位 医养结合
```

#### 第8章：城市定位与发展建议
- 补充与同级城市的横向对比数据，强化定位论据
- 充实文化、商业、旅游、产业四个维度的具体建议支撑案例
- 搜索引擎重点：必应国际、Google HK、人民网

```
{城市名称} 与 {同类城市} 对比 竞争力
{城市名称} 文旅 发展 规划 目标
{城市名称} 商业 发展 定位 战略
```

#### 第9章：发展展望与实施路径
- 补充相关规划文件中的量化目标（GDP、人口、产业占比等）
- 充实阶段性路径的政策依据和落地抓手
- 搜索引擎重点：政府官网、发改委、百度

```
{城市名称} 十四五 规划 目标 GDP 人口 2025
{城市名称} 发展规划 量化 指标 产业
{城市名称} 政府工作报告 2024 2025 目标
```

### 2.3 充实质量标准

每章充实完成后，须满足：
- 新增数据均已标注来源（机构名称）和时间节点（年份）
- 新增内容与原有文字风格完全一致（无列表符号、无表格、无emoji）
- 论证链条：主题句 → 数据/案例支撑 → 承转衔接
- 段落不少于150字

---

## 第三阶段：生成双格式输出文件

### 3.1 Markdown 文件输出

将完整最终报告写入 Markdown 文件，文件命名规则：

```
{城市名称}_调研报告_{YYYY-MM-DD}.md
```

文件保存到当前工作目录。

### 3.2 DOCX 文件输出

使用 `pandoc` 将 Markdown 文件转换为 DOCX 文件：

```bash
pandoc "{城市名称}_调研报告_{YYYY-MM-DD}.md" \
  -o "{城市名称}_调研报告_{YYYY-MM-DD}.docx" \
  --toc \
  --toc-depth=2
```

如果系统未安装 pandoc，使用以下备选命令安装后再转换：

```bash
# macOS
brew install pandoc

# Ubuntu/Debian
sudo apt-get install pandoc
```

文件命名规则：

```
{城市名称}_调研报告_{YYYY-MM-DD}.docx
```

### 3.3 输出完成确认

两个文件均成功生成后，向用户报告：
- Markdown 文件路径
- DOCX 文件路径
- 报告总字数（近似值）
- 各章节字数分布

---

## 注意事项

1. **子技能完整执行**：第一阶段必须完整执行 `city-research-deep` 的全部工作流程，不得跳过数据收集步骤
2. **充实不覆盖**：第二阶段充实时，只追加和扩展内容，不删除初稿已有内容，不改变原有文字风格
3. **数据一致性**：如充实阶段发现与初稿数据矛盾，以可信度更高的来源为准，并在报告中注明数据差异
4. **文件写入**：Markdown 文件须在第二阶段充实完成后一次性写入，确保是最终版本
5. **pandoc 转换**：DOCX 转换失败时，须向用户说明原因，并提供 Markdown 文件作为替代交付物
